## 🎨 UX 取捨（設計決策需要老闆裁決）

### Q4. 拖曳建立預約時，服務下拉是否「自動依時長篩選」？ `P0`

**背景**：老闆端日曆拖曳建立預約時，可以拖出時長（例如拖 1 小時 / 3 小時）。

**方案 A：自動篩選**
- 拖 1 小時 → 只顯示：男士剪髮 / 女士剪髮 / 護髮
- 拖 3 小時 → 只顯示：染髮 / 燙髮
- 拖 2 小時 → 顯示「無符合服務」，提示調整時長
- ✅ 選錯機率接近 0
- ❌ 無法做「特殊情況：染髮壓縮成 2 小時」這類破例

**方案 B：自由選服務 + 提示**
- 下拉顯示全部服務
- 選染髮（3h）但拖的是 1h → 跳提示「這個服務需要 3 小時，要自動延長嗎？」
- ✅ 保留彈性
- ❌ 多一步確認

**要問老闆**：實際營運中會不會有「為熟客壓縮或延長服務時間」的情境？

## 💰 定價與商業模式 `P0`

### M1. 訪談已知定價錨點
- 老闆自述可接受：NT$3,000–10,000/月訂閱
- 老闆認知：市面方案「好幾萬」→ 覺得太貴、只適合連鎖
- **gap 定位**：服務單店、價位合理

### M2. 要追問的定價問題
- 試用一個月 → 正式訂閱之間，有沒有「優惠綁年約」的空間？
- 價格是含 LINE 官方帳號費用（每月推播額度）還是另收？
- 開店前置費用（設定、客戶資料匯入）要不要收一次性 setup fee？

### M3. 願付價值的真實測試
- 「如果漏接 1 個染髮客 = 損失 NT$2,000+，系統幫你一個月少漏 3 個 → 月費 NT$X 你能接受？」
- 用具體金額反算而非模糊問「願意付多少」。

### M4. 客戶的價值感受
- 老闆自己認為有用 ≠ 客戶滿意。訪談：「客戶如果還是要回到 LINE 訊息就失敗」
- 要不要在上線前先訪談 3–5 位老顧客？他們願意改用新系統嗎？

## 📊 資料與觀察指標

### D1. 上線前要老闆提供的資料 `P0`
- 過去 3 個月手動紀錄的預約（Excel）—用來回填 CRM、觀察實際流量
- 老顧客名單（LINE ID / 電話）—用來一次性匯入

### D2. 要讓老闆看到什麼數字才覺得「值得每月付費」？ `P0`
- 候選指標：
  - 本月漏接次數（vs 上個月手動時期估計值）
  - 每日節省回訊時間（估算）
  - 違規 / No-show 數字（新價值：以前沒有）
  - 回流客戶分群（NEW / REGULAR / VIP / AT_RISK）
- 要問：**哪一個數字是你每週會打開來看的？** 那個就是主儀表板。

### D3. 「漏接 vs 不滿意」要分開觀察 `P1`
- 訪談：「基本上每天都可以排滿，只是我花太多時間在弄」
- 真正的價值不是「多接」，是「省時間」+ 「睡覺時也能預約」。
- 要驗證：系統上線後，老闆每天花在預約排程的時間 → 從多少分鐘 降到多少分鐘？
