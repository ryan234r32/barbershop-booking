# 老闆訪談逐字稿 — 2026-04-15（第二次訪談）

> 現場訪談，Ryan 把原型 demo 給老闆看，順便確認商業規則與痛點。
> 以下為逐字稿整理（語音轉文字，保留口語）。

## 一、營業基本資訊

- **公休**：禮拜三、禮拜日（之前一度說只有禮拜日，更正為「禮拜三 + 禮拜日」）
- **營業時間**：每天 11:00 – 20:00
- **時段單位**：1 小時（不做半小時）

## 二、服務項目與時長

| 類別 | 細項 | 時長（小時） | 備註 |
|---|---|---|---|
| 剪髮 | 男剪 / 女剪 / 學童剪 / 同剪 | 1 | 含洗頭 1000 元；不洗較少 |
| 燙髮 | — | 4 | 通常含護 |
| 染髮 | 補染 | 2 | |
| 染髮 | 全頭染 | 2.5–3（算 3） | |
| 漂髮 | 純漂（要退到乾淨色） | 3 小時／次，至少 3 次 | 很複雜 |
| 漂+染（調色） | 全流程 | 6 | **需要人腦判斷**，無法純勾選 |
| 瀏海修 / 護髮 / 西髮 | — | — | 附加項目，時間不固定 |

### 關鍵：染/漂的拆解討論

- 染跟護「幾乎都綁在一起」——客戶現在都是染+護，不接受只染不護
- 燙 → 基本只需吹龍頭、標準流程，OK 用勾選
- 染 → 也還好，可拆
- **漂**：客戶不一定知道漂完要不要染、想要什麼顏色 → **這一塊一定要人腦判斷**
- 共識：「把可以勾選的拆成 A/B/C/D，真的拆不了的（= 漂髮諮詢）就進『諮詢待回覆』流程」

## 三、預約行為觀察

- **排班偏好**：沒有特別偏好，不用刻意集中或分散；11–20 點每個整點都可能被約滿
- **預約開放期限**：
  - 原本想「半年」，討論後收斂為 **45 天**
  - 理由：老闆自己會在 1 個月前才排私人行程（例如小孩暑假出國），太久會跟客戶衝突
  - 例外：少數國外客會約 2–3 個月後 → 走「專人聯絡」流程，系統不直接放行事曆
- **確認機制**：
  - 45 天內 → 客戶預約後老闆手機一打開會跳出一筆一筆讓他按「已知道」
  - 45 天外（特殊、漂髮諮詢類）→ 浮在最上面需要親自回覆
  - 共識：「系統能解決的沉到下面，需要出馬的浮到最上面」

## 四、取消 / 改期政策

### 取消
- **24h 以上 → 線上直接取消**
- **24h 內 → 必須打電話**（增加取消成本）
- 點取消時引導「要不要改期」，很多情況會因此變成改期而非取消

### 改期
- 原本想 2 小時前才卡住，老闆想了之後：**全部開放線上改**（前一小時也可以）
- 理由：客戶都會提前 1 天以上改；當天很少，且當天有事通常會 LINE 溝通
- 觀察：大多數改期是文字溝通，很少打電話
- 共識：先全開放，若有人真的很誇張再針對性加規則（95–98% 客戶都 OK）

## 五、通知 / 訊息痛點（這是最大的問題）

### 現況流程

- 老闆在剪髮/染/燙當中不會看手機
- 客戶 LINE 問「可不可以約 X 時間」→ 老闆沒空回 → 客戶看到訊息後再回 → **一來一回常常拖 1-2 小時甚至隔天**
- 訊息可能包含：預約、諮詢、燙染問題、頭髮狀況照片
- 漂髮客戶會問超多問題（要不要漂、能不能燙、要拍照確認髮質）

### 老闆的訴求
- 「讓這一筆預約能夠更快確認，避免兩人溝通時間太久」
- 他剪一個客戶可能 30 分鐘有空檔，希望那空檔能一次處理完堆積的訊息
- **把『系統能解決的』自動化，『系統無法解決的』浮到最上面**

## 六、對帳（目前超麻煩）

### 現況
- 收現金 → 開 Excel 手動輸入
- 收轉帳 → 看銀行 App 跳通知 → 確認金額 → Excel 手動輸入
- 紅字 = 已轉帳到戶頭；黑字 = 現金（避免重複入帳）
- 每天晚上 8-9 點會再對一次

### Excel 欄位
新男剪、新女剪、新學童剪、童剪、新染、補染、心燙、瀏海漂、護髮、西髮、廠商、其他
（區分「新」是指 **這間店面**  的新客，不是 LINE 帳號新客。因為 3 年前從舊店搬過來。）

### 老闆偏好的收款流程
- 理想：預約結束前 20 分鐘，App 自動跳出「本次多少錢，顧客XXX」→ 按『現金』或『已轉帳』→ 完成
- 金流串接（1% 手續費）：老闆覺得「**1% 可以接受**」
- 過渡方案：顧客在 LIFF 按轉帳 → 自動跳出銀行帳號 → 顧客提供末五碼 → 老闆後台對帳核對

## 七、會員 / 分眾 / 行銷

### 「新客/舊客」定義要改
- 老闆的「新客」= 這間店面的新顧客（換店後計算）
- 不是 LINE 帳號的新註冊
- **需要支援「搬店後的舊客 → 新客分類」**

### CRM 分段（老闆 OK）
- NEW → REGULAR（60 天內 + 6 次以上）→ VIP（60 天內 + 12 次以上）
- AT_RISK（100 天沒來）→ LAPSED（半年沒來）
- 女生留長髮客可能 1 年才出現，屬於異常少數

### 行銷方案（Ryan 提案，老闆願意測試）
- 每個剪髮客戶剪完發 1 張「30 天內回來用的 9 折券」
- 剩 7 天推播提醒
- 一個月觀察 → 下個月開始測試看回訪率是否提升

### 儀表板重點
- **季度視角**（不是月，月太短看不出趨勢）
- 新客 vs 舊客比例 / 流失率
- 月營收、週營收、年營收
- 時段占用率（例：100 個時段中 14 個被預約 → 14%）
- 服務項目占比

## 八、支出 / 損益（新需求）

- 目前 Excel 開始加入私人支出，表比較亂
- 老闆希望系統能自動算收入，**也要能輸入支出** → 看「收入 - 支出 = 真實獲利」
- 固定支出（租金、水電）vs 變動支出（耗材、補貨） → 分兩類處理

## 九、常見問答自動化（老闆覺得很棒）

- 客戶 LINE 打到「燙」、「染」、「漂」這些關鍵字 → 自動跳方塊
- 方塊點下去 → 「請提供 A. 目前髮況照片 B. 想要的成品照片 C. XXX」
- 節省老闆每次重複打字

## 十、原型 Demo 反饋

- 行事曆時間格「想更窄一點」，希望一眼看到一整天
- 行事曆要能縮放（類似 Google Maps 手勢）
- 訊息介面：願意留在 LINE Official Account，但在 App 內整合也 OK（只要順手）

## 十一、其他瑣事

- 平板記錄為主（不用電腦）
- 一天高峰 5-6 點，週內前中後幾週忙閒不一定
- 六到七成滿為常態
