PM 工作原則 (Now / Next / Later)

  • Now 最多 3 件 — 限制 WIP，避免什麼都想做。
  • 先問為什麼 — 每張卡要有痛點、證據、下一步。
  • 先 Shape 再 Build — 模糊的東西先放 Next / Waiting，問清楚再進 Now。
  • 可靠性是產品力 — 試用前先有 timeout、fallback、checkpoint、救援步驟。
  • 指標刪除機制 — 月/年報每個指標問：老闆看到後能做什麼決策？答不出來就降級。

定期習慣 (Solo founder 節奏)

📊 Founder Score 怎麼算？

4 軸 × 25 分 = 100。Coach 不是 judge — 永遠針對最弱軸給 1 句具體下一步。

軸滿分條件0 分情境
WIP 自律NOW ≤ 3 件NOW ≥ 8 件（每超過 1 件扣 5 分）
出貨速度每週完成 ≥ 1.5 張卡片30 天 0 件
週期時間NOW → DONE 平均 ≤ 7 天平均 ≥ 30 天線性扣到 0
學習節奏連續 14 天有日記 entry連續 0 天

等級：85+ A / 70+ B / 50+ C / <50 D。卡片移到 NOW 自動記 startedAt，移到 DONE 自動記 doneAt。

不要做的事

  • 不要追求滿分 — 100 意味著太緊繃，留 15-20 分緩衝給意外
  • 不要為了刷分把卡片拆很碎 — 那是 gaming
  • 不要在月底壓力大時看分 — 退一步看趨勢

🛠️ GStack 節奏（Solo founder）

一句話：每天靠 preflight，每週靠 retro+health，每月靠 cso+learn，shipping 靠 review→ship→canary。

每日 ≤ 10 分

  • 開始：讀昨天日記最後一筆 Tomorrow（30 秒進入狀況）
  • commit 前：npm run preflight（typecheck+lint+test）
  • 收尾：跟 Claude 講「收工」會自動寫日記

每週日（30-60 分）

  1. /retro — 看本週 commits / 產出 / 趨勢
  2. /health — 程式碼健康分
  3. /triage — 升格 lessons-inbox
  4. /gstack-upgrade
  5. 寫 4 行 Weekly Retro：Shipped / Learned / Energy drain / Next week's bet

每月底（1-2 小時）

  1. /cso comprehensive — 深度安全稽核
  2. /document-release — 同步文件
  3. /learn — 整理累積的 lessons
  4. 重看月報，回應 P17：是否回答「哪裡偏掉？下個月調什麼？」

新功能 pipeline

/plan-eng-review → (高風險加 /codex) → 實作 → npm run preflight → /review → /ship → /land-and-deploy → /canary

修 bug pipeline

/investigate（先找根因，不直接修）→ fix → /review → /ship

📔 開發日記 5 行格式

每天 ≤ 5 分鐘，超過就不會持續。

欄位內容
Done1-2 件具體完成事，含檔案路徑/PR 號
Stuck/Decision卡點或決定，含 why
TIL今天學到的一件事 — 技術或商業都行
Tomorrow明天打開電腦的第一個動作（先讀 X，找 Y，目標 Z）
Mood一個字：順 / 卡 / 累 / 嗨 / 焦 / 平
