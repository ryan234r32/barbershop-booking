### 0.3 業界研究關鍵 takeaways
- 業界共識公式：**年營收 = 客戶數 × 年訪頻率 × 客單價**
- Bain：**+5% 留客率 = +25-95% 利潤**
- Strategies.com：年訪頻率 5 → 6 = **+20% 營收（沒任何新客）**
- 頂尖店家 60-70% 留客 *不靠折扣*，靠**一致 follow-up + rebooking**
- 9 折 30 天有效是合理 baseline，但**主動 outreach 才是真武器**

```typescript
interface Totals {

  bookings: number;
  revenue: number;
  uniqueCustomers: number;
  newCustomers: number;
  arpu: number;
  occupancyRate: number;
  cancellationRate: number;
  noShowRate: number;

  visitFrequency: number;          // 平均訪問次數/客戶/期間
  avgGapDays: number;              // 平均回訪間隔（天）
  medianGapDays: number;           // 中位回訪間隔（天）
  oneTimerRate: number;            // 一次性客戶比例 (%)
  retention90Days: number;         // 90 天回訪率 (%)
  retention60Days: number;
  retention30Days: number;
  yearlySpendPerCustomer: number;  // 平均每客戶年消費

  shopOldCustomers: number;        // 店面維度：舊店面客（從舊店跟過來）
}
```

**「店面維度」判斷邏輯**：
- 1008 搬家日期：(待老闆確認，先用 2024-12-31 當預設)
- 客戶第一次到「新店面」的日期 = User.firstVisitAt
- 如果 firstVisitAt < 搬家日期 → 舊店面客
- 如果 >= 搬家日期 → 新店面客
- 但 1169 筆 Excel 都是 2025（搬家後），所以這個算法在 Excel 資料上 = 全部都是「新店面客」← **bug**
- **真正算法**：應該看 Excel 的「新」前綴 — 服務名前綴「新」 = 客戶第一次來這間店面
- 改用 user 第一次預約的 service.notes 裡是否有「原始 Excel: 新X」來判斷

**回訪間隔算法**：
```
for each customer with >= 2 visits:
  sort visits by date
  for each consecutive pair (v_i, v_{i+1}):
    gap = (v_{i+1} - v_i) days
  add all gaps to global pool
return mean(gaps), median(gaps)
```

**留存率算法**：
```
for each customer:
  if first_visit + 90 days has another visit → retained_90
return retained_90 / total_customers * 100
```

### 4.3 Hero 估算公式

```typescript
function estimateRevenueIncrease(totals): { lower: number; upper: number; actions: string[] } {
  const baseline = totals.revenue;
  const oneTimerImprovement = (totals.oneTimerRate - 35) / 100; // gap to industry avg
  const retentionLever = baseline * Math.max(0, oneTimerImprovement * 0.4);
  // 把 1/2 一次性客拉回 = 增加 18% 客戶 × 平均年消費 NT$3876

  const utilizationGap = (75 - totals.occupancyRate) / 100;
  const utilizationLever = baseline * Math.max(0, utilizationGap * 0.6);
  // 拉到 65% 利用率 = +30% 產能

  return {
    lower: Math.round(retentionLever),
    upper: Math.round(retentionLever + utilizationLever),
    actions: [
      "啟用 30 天回訪推播",
      "Lapsed 客戶自動發 9 折券",
      "報表加空檔填充建議"
    ]
  };
}
```

→ 1008 baseline NT$135 萬 → 估算 +26-66 萬 = +19~49%。

### 6.2 動態到期日（按服務類型）— 用戶 insight

從用戶訊息：「**用平均回訪天數來設預約券**」

實作方式（3 個選項）：

| 選項 | 算法 | 優點 | 缺點 |
|---|---|---|---|
| (a) 全店單一天數 | 用 medianGap = 34 天 | 簡單 | 對染燙客太短 |
| (b) **按服務類型**（推薦） | 剪 30 天 / 染 60 天 / 燙 90 天 | 對齊頭髮自然週期 | 需要 service category 對應表 |
| (c) 個別客戶歷史 | 該客戶過去平均間隔 | 最精準 | 新客沒歷史 → fallback 服務類型 |

**建議實作 (b)**，留 (c) 為 V4：

```typescript
// src/lib/coupons/issue.ts
function getCouponDays(service: { name: string }): number {
  if (service.name.includes("剪")) return 30;
  if (service.name.includes("染") || service.name.includes("補染")) return 60;
  if (service.name.includes("燙") || service.name.includes("矯正")) return 90;
  if (service.name.includes("漂")) return 60;
  if (service.name.includes("護")) return 30;
  return 30; // default
}
```

→ 這也修改了 PRD-v3 §8 的 A/B test 定義（原本 30d vs 45d 分組）。新方案：**單一策略，按服務天數動態**。A/B 失去意義（除非要 A=固定30 vs B=動態服務）。
