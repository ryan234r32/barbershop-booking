## 1. 設計哲學

**「Apple Store 的現代極簡」×「Aesop 的植物有機感」**

營造高端、沉穩且值得信賴的精品沙龍氛圍。每個視覺決策都要問：
- 這個元素傳達「專業感」嗎？
- 這個元素讓人感覺「安靜」而非「喧鬧」嗎？
- 這個元素是必要的嗎？（如果不是，拿掉）

**情緒關鍵字：** 建築感 · 有機 · 安靜 · 高端 · 質地 · 呼吸感

**避免：** 霓虹色、俗氣漸層、高飽和度、卡通感、潮流感

**參考靈感：**
- **Apple Store** — 空間感、排版節奏、字體層次
- **Aesop** — 色溫、植物感、雜誌式排版
- **Kinfolk Magazine** — 大字距、大呼吸感
- **Muji** — 減法美學、材質感

## 2. 色彩系統

### 2.1 核心色票

| 角色 | 名稱 | HEX | CSS Token | 用途 |
|------|------|------|-----------|------|
| **品牌主色** | 深森林綠 | `#003D2B` | `--color-brand` | CTA 按鈕、選中狀態、標題、品牌 Logo |
| **背景主色** | 暖乳白 | `#FFF8F1` | `--color-bg` | 全域頁面背景、主要卡片底色 |
| **輔助背景** | 柔沙色 | `#F3ECE4` | `--color-surface` | 未選中卡片、次要區塊底色 |

### 2.2 文字色

| 角色 | HEX / Alpha | CSS Token | 用途 |
|------|-------------|-----------|------|
| **標題文字** | `#003D2B` | `--color-text-primary` | 大標題、強調（與品牌主色同色） |
| **內文** | `#2D3A30` | `--color-text-body` | 段落、副標、描述文字 |
| **輔助文字** | `rgba(0, 61, 43, 0.5)` | `--color-text-muted` | 提示、placeholder、meta |
| **停用文字** | `rgba(0, 61, 43, 0.3)` | `--color-text-disabled` | Disabled 狀態 |

### 2.3 狀態色（植物色系）

> 刻意選用降低彩度的「植物色系」變體，避免破壞整體沉穩氛圍。

| 用途 | 名稱 | HEX | CSS Token |
|------|------|------|-----------|
| 成功 / 可用 | 苔蘚綠 | `#4A7C59` | `--color-success` |
| 警告 / 注意 | 琥珀陶 | `#C88B3B` | `--color-warning` |
| 錯誤 / 違規 | 赤陶紅 | `#A84A3B` | `--color-danger` |
| 停用 / 不可選 | 柔沙色 | `#F3ECE4` | `--color-disabled` |

### 2.5 色彩使用原則

**必做：**
- 背景使用 `#FFF8F1` 暖乳白，**絕不使用純白 `#FFFFFF`**
- CTA 按鈕一律 `#003D2B` 深綠底 + `#FFF8F1` 暖乳白字
- 大面積留白，讓色彩有呼吸空間
- 狀態色選低彩度植物色系

**禁止：**
- 不用純白 `#FFFFFF`（太冷，失去紙張質感）
- 不用亮綠、螢光綠、淺綠（破壞沉穩感）
- 不用漸層（保持扁平、建築感）
- 不用高飽和色（與 Aesop 植物感衝突）
- 不用霓虹、卡通、trendy 色調

## 3. 字體與排版

### 3.1 字體家族

| 用途 | 首選字體 | 備援 |
|------|---------|------|
| 英數字 | **Manrope** | Inter、SF Pro |
| 中文 | **Noto Sans TC**（思源黑體） | PingFang TC、Microsoft JhengHei |

```css
font-family: "Manrope", "Noto Sans TC", -apple-system, "PingFang TC", sans-serif;
```

### 3.2 字體層級

| 層級 | 用途 | 字重 | 字距 | 行高 | 範例 |
|------|------|------|------|------|------|
| **Display** | 大標、Hero | 700 Bold | 0.1em | 1.2 | Landing page 主標 |
| **H1** | 頁面標題 | 700 Bold | 0.05em | 1.2 | 「預約成功」「我的預約」 |
| **H2** | 區塊標題 | 600 SemiBold | 0.03em | 1.2 | 「選擇服務」 |
| **H3** | 卡片標題 | 600 SemiBold | 0.02em | 1.4 | 服務名稱 |
| **Body** | 內文 | 400 Regular | 0 | 1.6 | 描述、說明 |
| **Caption** | 輔助文字 | 400 Regular | 0.02em | 1.4 | 時長、meta |
| **Label** | 雜誌感標籤 | 500 Medium | **0.15em** | 1.2 | 「STEP 01」「SERVICES」 |

### 3.3 排版原則

**大呼吸感（Generous Whitespace）：**
- Section 之間至少 32px 間距
- Card 內部 padding 至少 16-20px
- 按鈕 padding 垂直至少 14-16px
- 永遠不要為了塞內容而縮小間距

**雜誌感細節：**
- 輔助標籤常用**全大寫 + 字距放寬**（例：「STEP 01」而非「第一步」）
- 大標題增加字距創造氣度（0.05-0.1em）
- 數字保持 tabular numbers（等寬）確保對齊

**對齊規則：**
- 主要內容：左對齊
- 標題 + Hero：可置中
- 價格、時間、meta：右對齊

## 4. 組件設計

### 4.1 按鈕（Buttons）

**形狀：** 直角或微圓角（4-8px），**絕不使用大圓角**（> 12px）

| 類型 | 背景 | 文字 | 邊框 |
|------|------|------|------|
| **Primary** | `#003D2B` | `#FFF8F1` | 無 |
| **Secondary** | 透明 | `#003D2B` | `1px solid #003D2B` |
| **Disabled** | `#F3ECE4` | `rgba(0,61,43,0.35)` | 無 |

- 字重：SemiBold (600)，字距 0.05em
- Padding：16px 24px
- 陰影：無或極輕 `0 1px 2px rgba(0, 61, 43, 0.08)`

### 4.2 卡片（Cards）

| 狀態 | 背景 | 邊框 | 陰影 |
|------|------|------|------|
| **Default** | `#F3ECE4` | 無 | 無或 `0 1px 2px rgba(0,0,0,0.03)` |
| **Selected** | `#FFF8F1` | `2px solid #003D2B` | 無 |

- Border-radius：8-12px
- Padding：16-20px
- **靠色塊區分層次，不靠陰影**
- Selected 右上角可加小圓形 checkmark badge

### 4.3 輸入框（Inputs）

- 背景：透明或 `#F3ECE4`
- 邊框：底線式 `1px solid #003D2B`（無完整框線）
- 文字：`#2D3A30`
- Placeholder：`rgba(0, 61, 43, 0.5)`
- Focus：邊框加粗為 2px
- Padding：14px 16px
- 圓角：4-8px

### 4.4 圖示（Icons）

- **風格：線性（Outline），絕不使用實心或 3D**
- 線條粗細：均勻 1.5-2px
- 顏色：`#003D2B` 或 `#2D3A30`
- 尺寸：16px（inline）/ 20px（標準）/ 24px（標題旁）/ 32px+（Hero）
- 建議 library：Phosphor Icons (Regular) / Lucide
- 不使用 emoji 當主要圖示

### 4.5 Bottom Sheet

- 背景：`#FFF8F1`
- 頂部 drag handle：`#F3ECE4` 小橫條（40px × 4px）
- Border-radius：頂部 16px
- 背景遮罩：`rgba(45, 58, 48, 0.5)` 半透明暗色
- Padding：20-24px

### 4.6 Badge / Tag

- 背景：`#F3ECE4` 或透明
- 文字：`#003D2B`
- 字重：Medium (500)
- 字距：0.1em + UPPERCASE
- Padding：4px 10px
- Border-radius：4px 或 pill（999px）

## 5. 空間系統

### 5.1 Spacing Scale（4px 基底）

| Token | 值 | 用途 |
|-------|-----|------|
| `--space-1` | 4px | 極小間距（icon 與文字） |
| `--space-2` | 8px | 小間距（卡片內元素） |
| `--space-3` | 12px | 中小間距 |
| `--space-4` | 16px | 標準間距（卡片 padding） |
| `--space-5` | 20px | 標準間距（區塊內元素） |
| `--space-6` | 24px | 區塊間距 |
| `--space-8` | 32px | Section 間距 |
| `--space-10` | 40px | 大區塊間距 |
| `--space-12` | 48px | 頁面區塊分隔 |
| `--space-16` | 64px | 大頁面分隔 |

### 5.2 Border Radius Scale

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 4px | 小按鈕、tag |
| `--radius-md` | 8px | 按鈕、輸入框 |
| `--radius-lg` | 12px | 卡片（最大值） |
| `--radius-xl` | 16px | Bottom sheet 頂部 |

**絕不使用 radius > 20px（破壞建築感）**

## 6. 動效原則

### 6.1 可使用

- **微過場**：按鈕點擊 `scale(0.97)`，150ms ease-out
- **淡入淡出**：Bottom sheet 滑入、toast 出現，200-300ms
- **Skeleton loading**：柔和的 shimmer 動畫

### 6.2 禁止

- 彈跳動畫（bounce）
- 旋轉 spinner 單獨使用
- 誇張的 transform（rotate、skew）
- 過長的動畫（> 500ms）

**原則：** 動效應該是「發現得到但不被打擾」。

## 7. 視覺原則

### 7.1 雜誌感排版（Editorial Layout）
- 大面積色塊 + 文字對比
- 不對稱但有節奏的佈局
- 重要資訊用「留白 + 大字」凸顯

### 7.2 有機質地（Organic Texture）
- 暖色調背景取代冷白
- 植物色系狀態色
- 避免銳利的純黑、純白

### 7.3 建築感（Architectural）
- 直角與微圓角並用
- 清晰的區塊分界
- 理性的網格系統

## 8. Do & Don't

### DO
- 使用暖乳白 `#FFF8F1` 作為背景
- 大膽使用留白
- 選用線性圖示（outline）
- 標題加字距、小標用大寫
- 用色塊區分卡片而非陰影
- 狀態色選低彩度植物色系

### DON'T
- 不用純白 `#FFFFFF` 背景
- 不用大圓角（> 12px）
- 不用漸層
- 不用 drop shadow 堆疊層次
- 不用高飽和霓虹色
- 不用卡通化圖示或 emoji 作為主視覺

## 10. 快速 Checklist

設計任何新頁面或元件時，用這個 checklist 自我檢查：

**色彩：**
- [ ] 使用了 `#FFF8F1` 而非 `#FFFFFF`
- [ ] CTA 是 `#003D2B` 深森林綠
- [ ] 沒有使用漸層、霓虹、高飽和色

**排版：**
- [ ] 大量留白，不擁擠
- [ ] Section 之間至少 32px 間距
- [ ] 主標題有字距（tracking）增加氣度

**組件：**
- [ ] 按鈕是直角或微圓角（4-8px）
- [ ] 卡片沒有重陰影
- [ ] 圖示是線性（outline）風格

**整體：**
- [ ] 感覺「安靜」而非「喧鬧」
- [ ] 感覺「精品」而非「trendy」
- [ ] 感覺「建築感」而非「可愛」
