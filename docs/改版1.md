# 1008 Hair Studio — LINE 預約系統優化研究報告

## 一、顧客旅程分析：從加好友到完成預約

### 目前的問題
顧客加好友後，進入對話框只看到一片空白（沒有 Rich Menu）和一句固定回覆文字。他完全不知道這個帳號可以幫他預約，也不知道怎麼開始。等於你花了力氣把人帶進店裡，結果店裡沒有招牌、沒有菜單、沒有服務員。

### 理想的顧客旅程（6 個接觸點）

```
加好友 → 歡迎訊息（Flex Message 卡片）→ Rich Menu 常駐底部
  ↓
點「立即預約」→ LIFF 預約頁面 → 選服務/日期/時段 → 確認
  ↓
收到預約確認訊息（Flex Message）→ 預約前 24h/1h 提醒
  ↓
到店消費 → 消費後感謝訊息 → 引導回預約（下次見）
```

每一個接觸點都要讓顧客覺得「很簡單、很清楚、不用想」。

## 二、歡迎訊息設計（優先級 #1）

### 業界最佳實踐

根據 LINE 官方的美業指南和 metabiz 的案例分析，歡迎訊息是「第一印象」，決定了顧客會不會繼續互動。好的歡迎訊息要在 3 秒內讓人知道：你是誰、你能幫我做什麼、我下一步該怎麼做。

### 推薦設計：2 則訊息組合

**第 1 則：文字歡迎（親切感）**
```
{好友名稱} 你好！🙌
歡迎來到 1008 Hair Studio ✂️

我們位在台北中正區新生南路
週二～週日 11:00-20:00 為您服務

👇 點下方選單就能輕鬆預約！
```

**第 2 則：Flex Message 行動卡片（引導行動）**

設計一張精美的 Flex Message 卡片，包含：
- 頂部：店面形象照片或品牌 Logo
- 中間：一句 slogan，例如「專屬你的風格，一鍵預約」
- 底部 2 個按鈕：
  - 🗓 **立即預約**（打開 LIFF 預約頁面）
  - 📋 **查看服務與價格**（打開服務列表頁或觸發關鍵字回覆）

### 為什麼用 Flex Message 而不是普通文字？

Flex Message 可以做出類似 App 介面的互動卡片，有按鈕、有圖片、有排版。相比純文字，點擊率高 3-5 倍。LINE 官方把它形容為「訊息樂高」——自由度極高，可以做出媲美 App 的視覺效果。

### 技術實作方式

透過 Messaging API 的 Webhook，在 `follow` 事件（加好友）時發送 Flex Message。你的系統已經有 webhook handler（`/api/webhook`），只需要在 follow 事件裡加上 Flex Message 的 JSON 就可以了。

## 三、Rich Menu 設計（優先級 #2）

### 業界洞察

根據 LINE 官方數據，設計良好的 Rich Menu 能減少 60-80% 的重複性客服問題（價目表、地址、營業時間）。它是 LINE 帳號裡「唯一免費的常駐曝光版位」，浪費它等於浪費錢。

### 推薦版型：2×3 六格（大尺寸）

```
┌─────────────┬─────────────┐
│   🗓️         │   📋         │
│  立即預約    │  我的預約    │
│             │             │
├─────────────┼─────────────┤
│   💇         │   📍         │
│ 服務與價格   │ 店家資訊     │
│             │             │
├─────────────┼─────────────┤
│   📸         │   📞         │
│ 作品集      │  聯絡我們    │
│             │             │
└─────────────┴─────────────┘
```

### 每格的功能對應

| 按鈕 | 動作類型 | 實際行為 |
|------|---------|---------|
| 立即預約 | URI | 打開 LIFF 預約頁面 `https://liff.line.me/2009556442-ipw42ffU` |
| 我的預約 | URI | 打開 LIFF 我的預約頁面 |
| 服務與價格 | 文字 → 關鍵字觸發 | 送出「服務價格」→ 觸發 Flex Message 顯示價目表卡片 |
| 店家資訊 | URI | 打開 Google Maps 或顯示地址/營業時間 |
| 作品集 | URI | 連結到 Facebook 或 Instagram |
| 聯絡我們 | 文字 | 送出「我要聯絡店家」→ 提示老闆會在空檔回覆 |

### 視覺設計原則

根據多個成功案例歸納：
- **背景乾淨 + 小 Icon + 主題字**：這是失敗率最低的設計公式
- **配色統一**：建議用莫蘭迪色系或黑白金配色（符合理髮廳高質感定位）
- **最重要的功能放左上角**：用戶視線習慣是 Z 字型掃描，左上角點擊率最高
- **每個按鈕一目了然**：讓人一看就知道「點了會發生什麼」

### 進階：分眾 Rich Menu（V2 可做）

未來可以做「新客選單」vs「會員選單」：
- **新客**：「認識我們」「首次優惠」「立即預約」
- **會員**：「立即預約」「我的預約」「集點查詢」

這需要用 Messaging API 的 Rich Menu API 來實現，不是 LINE OA Manager 內建的功能。但技術上你的系統已經有 CRM 分群，完全可以根據 NEW/REGULAR/VIP 切換不同選單。

### 技術實作方式

Rich Menu 有兩種做法：
1. **LINE OA Manager 後台設定**（簡單，不需寫程式，但功能有限）
2. **Messaging API 程式控制**（靈活，可以做分眾切換，但需要寫程式）

Demo 階段建議先用方法 1 快速上線，之後再用方法 2 做分眾。

## 四、關鍵字自動回覆（優先級 #3）

### 推薦設定的關鍵字

| 關鍵字 | 回覆內容 | 格式 |
|--------|---------|------|
| 預約 / 我要預約 / 訂位 | 引導到 LIFF 預約頁面的 Flex Message | Flex Message + 按鈕 |
| 服務 / 價格 / 價目表 / 多少錢 | 完整價目表 Flex Message（輪播卡片） | Flex Carousel |
| 時間 / 營業時間 / 幾點 | 營業時間 + 地址 + Google Maps 連結 | Flex Message |
| 取消 / 改時間 / 更改 | 引導到「我的預約」頁面 | Flex Message + 按鈕 |
| 地址 / 在哪 / 怎麼去 | 地址 + Google Maps 連結 + 交通方式 | Flex Message |
| 你好 / 哈囉 / Hi | 簡短歡迎 + 提示用底部選單 | 文字 |

### 價目表 Flex Message 設計建議

用 **Carousel**（輪播卡片）格式，每張卡片一個分類：

```
[卡片 1: 剪髮]         [卡片 2: 染髮]        [卡片 3: 燙髮]
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ ✂️ 剪髮       │  │ 🎨 染髮       │  │ 💫 燙髮       │
│              │  │              │  │              │
│ 男性 $1,000  │  │ 補染 $2,200  │  │ 溫塑燙 $4,000│
│ 女性 $1,100  │  │ 漂髮 $2,600↑ │  │ 縮毛矯正     │
│              │  │ 染髮 $2,600↑ │  │      $4,600↑ │
│              │  │              │  │              │
│ [立即預約]   │  │ [立即預約]   │  │ [立即預約]   │
└──────────────┘  └──────────────┘  └──────────────┘
→ 左右滑動查看更多
```

每張卡片底部都有「立即預約」按鈕，直接跳到該服務的預約流程。

### 技術實作方式

你的系統已有 webhook handler，只需要在收到 `message` 事件時做關鍵字比對，然後回覆對應的 Flex Message JSON。可以用簡單的 includes() 做模糊比對，不需要 NLP。

## 五、LIFF 預約頁面 UI 優化（優先級 #4）

### 針對一人店的流程簡化

1008 是一人店，不需要「選設計師」步驟。目前的 5 步流程（選服務→選日期→選時段→確認→成功）對一人店來說已經很合理。

### 可優化的點

1. **首頁品牌感**：進入 LIFF 後先看到店家 Logo + 一句歡迎語，而不是直接跳到服務列表
2. **服務卡片化**：每個服務用卡片展示（名稱 + 價格 + 預估時間 + 簡短描述），而不是純列表
3. **時段視覺化**：用顏色區分可預約/已滿/即將額滿，讓顧客一眼看到熱門時段
4. **確認頁資訊完整**：顯示服務 + 日期 + 時間 + 價格 + 地址 + 取消政策提醒
5. **成功頁引導**：預約成功後顯示「加入行事曆」按鈕 + 分享按鈕 + 「回到 LINE」按鈕

### 參考案例：inline（愛排）的做法

inline 雖然是餐廳預約系統，但他們的 UX 原則值得參考：
- 極簡步驟（3 步內完成）
- 每步只問一個問題
- 視覺回饋明確（選中的項目有明顯高亮）
- 預約完成後有清楚的確認資訊和行事曆整合

## 七、給老闆 Demo 的策略建議

你說想讓老闆覺得「很專業」同時顧客覺得「很方便」。Demo 的時候建議這樣展示：

### 展示順序（老闆視角）

1. **先演示顧客體驗**：用手機加好友 → 看到歡迎訊息 → 點 Rich Menu → 30 秒完成預約 → 收到確認訊息。讓老闆從「顧客角度」感受便利性。

2. **再演示管理後台**：打開電腦 → Dashboard 一目了然今天有幾個預約 → 點開行事曆看排程 → 客戶資料自動建檔 → CRM 分群。讓老闆覺得「再也不用手動記 Excel」。

3. **最後說痛點**：「你現在每天花多少時間回覆 LINE 訊息確認預約？」→ 「這個系統可以幫你省下這些時間，而且不會漏單。」

### 關鍵話術

對一人店老闆最有殺傷力的一句話：
> 「你一個人又要剪頭又要回訊息，客人傳訊息來你在忙根本看不到。這個系統讓客人自己約，你專心做你的手藝就好。」

## 八、競品差異化：你的系統 vs 美業歐巴 / ezPretty

| 比較項目 | 你的系統 | 美業歐巴 | ezPretty |
|---------|---------|---------|----------|
| 月費 | 可自訂（你定價） | ~$2,000-5,000/月 | ~$1,500-3,000/月 |
| LINE 整合深度 | 深（LIFF + Bot + Rich Menu + CRM 全包） | 中（主要是預約） | 淺（只是連結跳轉） |
| 客製化 | 完全客製 | 模板化 | 模板化 |
| CRM | 自動分群 + 行為追蹤 | 基本 | 無 |
| 技術門檻 | 需要你維護 | SaaS 免維護 | SaaS 免維護 |

你最大的差異化是 **LINE 原生整合深度** + **完全客製化能力**。美業歐巴和 ezPretty 都是通用型 SaaS，無法針對單一店家做深度客製。你可以做到「顧客從頭到尾不離開 LINE」的體驗，這是它們做不到的。
